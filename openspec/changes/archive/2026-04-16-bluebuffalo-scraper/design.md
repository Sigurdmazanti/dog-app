## Context

Blue Buffalo uses a Vue/PrimeVue-rendered tab layout on product pages. Each product has three tabs — Feeding Guidelines, Guaranteed Analysis, and Ingredients — rendered as accessible `[role=tab]` buttons and `[role=tabpanel]` panels. Tab panel IDs are dynamically generated (e.g. `pv_id_0_tabpanel_1`) and are not stable across page loads or products. The page structure is otherwise static HTML delivered server-side, so Cheerio is sufficient — no headless browser is needed.

## Goals / Non-Goals

**Goals:**
- Extract a composed product title from `<h1>` (sub-brand) and `<h3 itemprop=name>` (variant), with trademark symbols stripped from the h1
- Extract the full ingredients list from the Ingredients tabpanel as a comma-joined description string
- Extract the Guaranteed Analysis table rows from the Guaranteed Analysis tabpanel as a formatted text string
- Register `bluebuffalo.com` in the source registry
- Provide a source YAML and selector reference following existing conventions

**Non-Goals:**
- Fetching or paginating product listing pages (product URLs are provided manually)
- Parsing or validating calorie content from the Feeding Guidelines tab
- Handling multi-region variants of the site

## Decisions

### Decision: Locate tabpanels via tab button `aria-controls`, not by positional index

The tabpanel IDs (`pv_id_0_tabpanel_N`) are generated by PrimeVue and are not guaranteed stable. Rather than selecting by ID directly or by `:nth-child`, the scraper finds the `[role=tab]` button whose text matches the desired section label, reads its `aria-controls` attribute, and then selects the corresponding `[role=tabpanel]`.

**Alternatives considered:**
- Select by index (e.g. second tabpanel) — fragile if tab order changes
- Select by hardcoded ID — would break if PrimeVue changes its counter prefix

### Decision: Reconstruct ingredients list from `data-tag-text` attributes and `<span>` text

Ingredients in the Ingredients tabpanel are rendered as a mix of `<a class="Modal-trigger Tag" data-tag-text="...">` links (ingredients with modal detail pages) and plain `<span>` elements (ingredients without a detail page). To preserve order, the scraper iterates all direct children of the wrapper div, extracting `data-tag-text` from anchors and text content from spans, then joins them with `, `.

**Alternatives considered:**
- Use `.text()` on the entire panel — includes button text and whitespace noise
- Select only `a[data-tag-text]` — silently drops span-only ingredients

### Decision: Format Guaranteed Analysis as `Name: Value` lines joined by newline

The analysis table uses `<th class=as-td>` for nutrient names and `<td>` for values. Each row is formatted as `Name: Value` and the rows are joined with `\n` to produce a consistent text block for downstream AI composition mapping.

## Risks / Trade-offs

- [PrimeVue ID prefix changes] → Mitigation: selector relies on `aria-controls` attribute rather than hardcoded IDs; robust to counter resets
- [Ingredient spans vs anchors ordering] → Mitigation: iterate all children in DOM order rather than querying each type separately
- [Trademark symbols in h1] → Mitigation: strip `™` and `®` using a regex before composing the title
